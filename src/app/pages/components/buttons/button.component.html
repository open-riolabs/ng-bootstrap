Buttons

Use Bootstrap’s custom button styles for actions in forms, dialogs, and more with support for multiple sizes, states, and more.

Base directive.

You can use the rlb-button directive to create a button with different colors and sizes. Without any additional configuration, it will render a primary button.

Variants.

Bootstrap includes several button variants, each serving its own semantic purpose, with a few extras thrown in for more control.

Button Types.

The rlb-button directive is primarily intended to be used with the native <button> element. However, it can also be applied to <a> and <input> elements to ensure consistent styling across different types of interactive controls. Be aware that some browsers may apply slightly different rendering depending on the element used.

When using an <a> element styled as a button to trigger in-page actions (such as toggling content, opening modals, or executing JavaScript logic), rather than navigating to a different page or section, you should add role="button". This helps assistive technologies, such as screen readers, correctly interpret the element’s purpose.

Outline buttons

Need a button without the default background colors? You can use the outline directive to display a button with an outlined style instead of a solid background.

When the outline directive is applied, the button is rendered without background colors or background images, keeping only the border and text color. This works with all buttons using the rlb-button directive.

HTML

Sizes

Use the size attribute to control the button size. Supported values are sm, md, and lg. If size is not provided, md is applied by default.

HTML

Disabled state

Make buttons appear inactive by adding the disabled boolean attribute. Disabled buttons have pointer-events: none applied, preventing hover and active states from being triggered.
